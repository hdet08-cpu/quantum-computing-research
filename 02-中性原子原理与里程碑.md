# 中性原子量子计算 · 底层原理与发展里程碑

## 一、底层原理

### 1.1 为什么叫"中性原子"

中性原子不带电，区别于"离子阱"路线（用带电的离子）。不带电意味着：
- **优势**：原子之间基态时几乎没有相互作用，可以在一个真空腔里放上千甚至上万个，互不干扰
- **难点**：电场对它不起作用，需要用激光来捕获和操控

### 1.2 怎么做量子比特

取一个原子（通常是铷-87、铯-133 或锶-88），选其最外层电子的两个稳定低能级作为 |0⟩ 和 |1⟩。

这两个能级位于原子的**超精细结构（hyperfine structure）**中——同一个基态因为电子自旋和原子核自旋的耦合，分裂成能量略有不同的两个子能级。因为是基态，非常稳定，**相干时间可达秒级**（超导是微秒级，差了 6 个数量级）。

### 1.3 怎么抓住原子：光镊与光晶格

#### 光镊（Optical Tweezers）— 主流方案
一束高度聚焦的激光在焦点处形成微型"光阱"。原子在激光场中被极化，产生指向光强最大处的力，把原子拉向焦点。通过调节激光焦点位置，可以随意移动原子。

> 类比：用激光做了一个微型弹珠台的凹坑，原子滚进坑里就被困住了。

**使用者**：QuEra、Pasqal、Atom Computing、原子矩阵

#### 光晶格（Optical Lattice）— planqc 的独特路线
两束反向激光形成驻波，产生周期性亮暗条纹。原子坐在每个亮点（或暗点），形成整齐的二维或三维网格。

| 维度 | 光镊 | 光晶格 |
|---|---|---|
| 灵活性 | 可任意排列、可移动 | 位置固定 |
| 规模化 | 需要复杂光学系统 | 天然大规模 |
| 使用者 | QuEra, Pasqal, Atom Computing | planqc |

### 1.4 激光冷却

室温下原子以几百米/秒乱飞，根本抓不住。需要先用**激光冷却**：
1. 让激光频率略低于原子共振频率
2. 根据多普勒效应，只有向激光方向运动的原子才能吸收光子并被减速
3. 六束交叉激光从六个方向照射，原子无论往哪飞都被减速
4. 最终"冻结"到微开尔文（~10μK）

冷却后装载进光镊/光晶格。装载是概率性的——拍照检查哪些阱有原子、哪些空，然后用光镊重排成无缺陷阵列。

### 1.5 量子门操作

#### 单比特门
用微波或激光脉冲精确地让一个原子在 |0⟩ 和 |1⟩ 之间转换。保真度已达 99.98%+。

#### 双比特门 — 里德堡态是核心

**里德堡态（Rydberg State）**是理解中性原子量子计算的最核心概念。

正常情况下中性原子之间几乎没有相互作用。解决方案：把原子的外层电子激发到**非常高的能级**（主量子数 n ~ 50-100）。此时电子轨道半径极大，原子变成巨大的"电偶极子"。两个膨胀的原子之间产生很强的**范德华相互作用**，作用距离可达几微米。

> 类比：正常原子像紧握的拳头，彼此碰不到。里德堡态的原子像张开双臂的人——两人之间就能握手互动了。但手臂张开是不稳定的（~100μs 衰变回来），所以操作必须足够快。

#### 里德堡阻塞（Rydberg Blockade）— 门机制

当两个原子靠得很近时，如果一个已经处于里德堡态，强相互作用会"阻止"邻近原子也被激发——因为双里德堡态的能量太高，激光不再共振。

**CZ 门（Controlled-Z Gate）的物理过程：**
- A=|0⟩：不激发，B 不受影响
- A=|1⟩, B=|0⟩：A 激发但 B 不参与，无事发生
- A=|1⟩, B=|1⟩：A 激发后阻塞 B → B 获得额外相位翻转

结果：只有两个都是 |1⟩ 时波函数获得负号 = 受控-Z 门。

#### 门保真度的误差来源
1. **里德堡态自发衰变**：寿命 ~100μs，门操作 ~1μs，约 1% 概率衰变
2. **原子热运动**：即使 μK 温度仍有微小振动，影响原子间距和相互作用强度
3. **激光相位噪声**：频率/相位抖动导致操作不精确
4. **原子丢失**：里德堡态原子不再被光镊牢固束缚

### 1.6 原子选择

| 原子 | 特点 | 使用者 |
|---|---|---|
| **铷-87 (Rb)** | 最成熟，冷却技术最完善 | QuEra, Pasqal, Atom Computing, 原子矩阵 |
| **铯-133 (Cs)** | 里德堡态寿命更长 | Infleqtion |
| **锶-88 (Sr)** | "钟跃迁"态，相干时间极长，但激光更复杂 | planqc, Atom Computing |
| **镱 (Yb)** | 更快门速度、可擦除误差、通讯波段互联 | 太一量生, Atom Computing |

### 1.7 量子纠错

#### 为什么需要纠错
保真度 99.9% 看起来很高，但运行 1000 次门操作后几乎必然出错。实用算法需要百万到十亿次门操作。解决方案：用多个物理比特"投票"保护一个逻辑比特。

#### 关键概念

- **Surface Code（表面码）**：最主流纠错方案，物理比特排成二维网格。典型编码比 ~100:1
- **码距（Code Distance, d）**：纠错码能纠正多少错误。码距 d 可纠正 ⌊(d-1)/2⌋ 个错误
- **阈值（Threshold）**：物理错误率的临界值。低于阈值时，增加码距可指数级降低逻辑错误率
- **编码比**：物理比特 : 逻辑比特。越小越好。Quantinuum 2:1（惊人），QuEra ~4.7:1，传统 surface code ~100:1

#### 中性原子纠错的独特优势

**擦除错误（Erasure Error）**：原子丢失时你**知道**哪个丢了（拍照发现空位）。已知位置的错误远比未知位置的错误容易纠正，理论上纠错开销降低 2-3 倍。

**可重构连通性**：光镊移动原子，让任意两个靠在一起做门，不受物理布局限制。可高效实现 surface code、color code、toric code 等多种纠错码。

**横向门（Transversal Gate）**：两个码块物理对齐后同时做门，天然不扩散错误，比"魔术态蒸馏"开销小 10-100 倍。

## 三、中性原子 vs 其他路线

| 维度 | 中性原子 | 超导 | 离子阱 | 光量子 |
|---|---|---|---|---|
| 比特规模 | ★★★★★ | ★★★ | ★★ | 尚未演示计算 |
| 门速度 | ★★ (~1μs) | ★★★★★ (~10ns) | ★★ (~100μs) | ★★★★ |
| 门保真度 | ★★★★ (99.94%) | ★★★★ (99.91%) | ★★★★★ (99.99%) | N/A |
| 连通性 | ★★★★★ 可重排 | ★★ 最近邻 | ★★★★★ 全对全 | 取决于光路 |
| 相干时间 | ★★★★★ 秒级 | ★★ μs级 | ★★★★ 秒级 | N/A |
| 纠错进展 | ★★★★★ 96逻辑 | ★★★ 1个 | ★★★★ 48-94 | ★★★ 12 GKP |
| 工作温度 | 真空室 | 15mK 极低温 | 室温真空 | 部分室温 |

**核心优势**：可扩展性 + 可重构连通性 + 纠错友好
**核心劣势**：单门速度慢（但可并行补偿）、门保真度略低于离子阱
